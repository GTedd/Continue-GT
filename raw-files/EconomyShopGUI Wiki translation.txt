[size=5][b]EconomyShopGUI - 概览[/b][/size]
这里是 EconomyShopGUI 的维基页面。了解、安装并学习如何配置和使用本插件！

[size=4][b]什么是 EconomyShopGUI（ESGUI），它可以做什么？[/b][/size]
这是一个完全可自定义的 GUI 商店插件，玩家可在这里向服务器购买或出售物品。插件特色丰富且配置详细：例如等级事件、加入信息、玩家交易记录以及更多。[配置文本在这里浏览]最好的部分是你可以完全自定义商店中的每一个物品[商店配置在这里浏览]在玩家不同等级的价格！

[size=3][b]核心功能：[/b][/size]

[list]
[*]高度自定义性；
[*]通过游戏内命令添加/编辑/删除商店内的物品；
[*]按你需要添加商店内的物品分类。例如：方块、红石、药水、食物、掉落物等；
[*]支持设置商店分类选择界面；
[*]向商店添加无限多的物品；[i]*1[/i]
[*]通过权限向商店添加折扣和/或出售翻倍；
[*]按物品或商店设置季度价格；
[*]通过子区经济来设置单次购买使用的多种货币；
[*]支持通过 Discord SRV 将玩家交易记录发送至 Discord 聊天群组；
[*]添加不同的山羊号角、附魔、药水效果、刷怪笼种类、知识之书、玩家头颅、烟花火箭和 RGB 皮革盔甲；
[*]通过 Geyser 支持基岩版；
[*]支持 SilkSpawners、WildSacker、UltimateStacker、RoseStacker、MineableSpawners 和 EpicSpawners 等第三方刷怪笼插件兼容；
[*]通过选项物品/商店界面或填充物品来修改商店的外观；
[*]多语言支持；
[*]设置禁用商店的游戏模式；
[*]支持 NPC 商店；
[*]多经济支持；
[*]用于追踪服务器经济流向的交易记录日志文件；
[*]在玩家购买或出售物品的时候执行特定的命令；[i][color=Orange]*仅付费版[/color][/i]
[*]将带有自定义 NBT 数据的物品添加至商店；[i][color=Orange]*仅付费版[/color][/i]
[*]基于物品的经济，可通过物品形式的货币与商店交易；[i][color=Orange]*仅付费版[/color][/i]
[*]允许设置物品不同的出售/收购价格，以及出售/收购物品的不同方式；[i][color=Orange]*仅付费版[/color][/i]
[*]按玩家全局限购物品；[i][color=Orange]*仅付费版[/color][/i]
[*]动态定价允许物品保持正常供需关系；[i][color=Orange]*仅付费版[/color][/i]
[*]在物品自定义名称/描述、商店物品和界面中的 PlaceholderAPI 完全支持；[i][color=Orange]*仅付费版[/color][/i]
[*]直接支持 EcoEnchants、AdvancedEnchantments、ExcellentEnchants 注册的插件附魔；[i][color=Orange]*仅付费版[/color][/i]
[*]通过 MySQL 在服务器之间同步限购或动态定价。[i][color=Orange]*仅付费版[/color][/i]
[/list]

[size=4][b]开始使用 ESGUI：[/b][/size]

ESGUI 由 4 个可完全配置的文件构成，这些文件存储着插件的一切东西，包括商店中的物品。除非你只使用命令来配置商店，否则你应该打开配置文件并学习它们是如何在服务器中运行的。[i]本插件中的命令也与配置文件的结构有一定关联。[/i]

[list=1]
[*]config.yml - 这里你可以配置 ESGUI [i]如何[/i]运作。任何对配置文件的改动都需要你重启服务器来生效；
[list=0]
[*]该文件的更多信息可以在这里查询！
[/list][*]sections.yml - 输入 [font=Consolas]/shop[/font] 命令第一次打开的界面，即主菜单配置所在区域。
[list]
[*]该文件的更多信息可以在这里查询！
[/list][*][font=Consolas]LanguageFiles[/font] 文件夹下的语言文件。
[list]
[*]（使用的语言）取决于你在配置文件 config.yml 中让 ESGUI 使用的语言，此处可修改 ESGUI 界面中的消息文本和其他提醒；
[*]该文件的更多信息可以在这里查询！
[/list][*]你需要安装前置插件，并为玩家设置相应的权限。
[/list]
[size=3][b]依赖插件：[/b][/size]

[list=1]
[*][b]经济插件[/b] - 你需要安装一个受支持的经济插件。目前本插件支持这些经济插件：
[list]
[*][url=https://www.spigotmc.org/resources/vault.34315/][b]Vault[/b][/url] - 在使用 Vault API 的情况下你仍然需要安装其他插件，例如 [url=https://www.spigotmc.org/resources/essentialsx.9089/]EssentialsX[/url] 或 CMI（使用注入插件）。
（译者注：Vault 不是经济插件，只是一个提供相关操作的经济接口）
[*][url=https://www.spigotmc.org/resources/gemseconomy.19655/][b]GemsEconomy[/b][/url] - 提供多经济支持。
[*][url=https://www.spigotmc.org/resources/playerpoints.80745/][b]PlayerPoints[/b][/url] - 这个经济插件支持 MySQL。
[*][url=https://www.spigotmc.org/resources/ultra-economy.83374/][b]UltraEconomy[/b][/url] - 这个插件同样支持多经济，且支持使用 MySQL 进行数据存储。
[/list][*][b]权限插件[/b] - 若你需要给予非 OP 玩家打开商店的权限，那么你就需要安装一个权限插件，见使用权限插件的教程章节。
[/list]

[*]注意：可选参数 [font=Consolas][玩家名称][/font] 只需在控制台中指定。

[*][font=Consolas][b]/reload[/b][/font] = 重载插件和商店（sections.yml 和 shop.yml）。*部分在 config.yml 下的设置需要重启服务器才可生效！
[*][font=Consolas][b]/shopgive <选区> <序号> [玩家名称][/b][/font] = 允许管理员直接从商店中获取某个物品并给予自己或其他玩家用于测试
[*][font=Consolas][b]/editshop additem <选区> <材料> <购买价格> <出售价格> [显示名称][/b][/font] = 通过该命令向商店添加一个物品
[*][font=Consolas][b]/editshop edititem <选区> <序号> <操作> <键> <值>[/b][/font] = 允许编辑商店内的已有物品
[*][font=Consolas][b]/editshop deleteitem <选区> <序号>[/b][/font] = 该命令允许你从商店中删除一个物品
[*][font=Consolas][b]/editshop import Essentials worth.yml[/b][/font] = 从 Essentials 导入所有物品的出售价格
[*][font=Consolas][b]/editshop addhanditem <选区> <购买价格> <出售价格>[/b][/font] = 在该命令的帮助下你可以将手上的物品添加至商店。（对包含隐藏/特殊 NBT 数据的物品很有用！）
[*][font=Consolas][b]/sellgui [玩家名称][/b][/font] = 该命令将会打开一个空白界面，你可以将物品放入其中以出售，关闭界面将会开始出售操作，给予玩家所得的同时返还不能出售的物品
[*][font=Consolas][b]/editshop addsection <选区> <图标材料名称> <显示名称> <位置>[/b][/font] = 允许你添加新的商店选区。
[*][font=Consolas][b]/editshop editsection <选区> <操作> <键> <值>[/b][/font] = 允许你编辑已存在的商店选区设置。
[*][font=Consolas][b]/editshop deletesection <选区>[/b][/font] = 从 sections.yml 配置中将指定的选区删除

[size=4][b]使用编辑商店命令：[/b][/size]
商店编辑命令是一种在游戏内快速更改商店的方法。

[list]
[*]在 3.0 以上的版本，你只能添加/编辑/删除商店中的物品，[i]它不能添加/编辑/删除任何的商店选区[/i]，你需要修改 sections.yml 才可以这么做。
[*]商店编辑命令给予 shops.yml 的格式，推荐你自行阅读该文件的解读，以便更快熟悉插件功能！
[/list]
[size=3][b]添加物品：[/b][/size]
将物品添加至商店只需输入命令 [font=Consolas][b]/editshop additem <选区> <材料> <购买价格> <出售价格> [显示名称][/b][/font]。

[list]
[*]在添加物品后你可能需要输入 [font=Consolas][b]/sreload[/b][/font] 才能让新添加的物品出现在商店中。
[/list]
[b]命令分解[/b]
[list]
[*][font=Consolas][b]/editshop additem <选区> <材料> <购买价格> <出售价格> [显示名称][/b][/font]
[*][font=Consolas][b]<选区>[/b][/font] = 所要添加物品的选区
[*][font=Consolas][b]<材料>[/b][/font] = 物品的材料名称
[*][font=Consolas][b]<购买价格>[/b][/font] = 物品的购买价格
[*][font=Consolas][b]<出售价格>[/b][/font] = 物品的出售价格
[*][font=Consolas][b][显示名称][/b][/font] = [i][可选参数][/i]物品的显示名称。可以包含空格、“&”彩色字符以及特殊字符。
[*]* 物品的[i][b]序号[/b][/i]在成功添加时会自动分配。
[*]* 额外设置（大多可选，少数必须设置）可能需要通过 [font=Consolas]edititem[/font] 命令设置（例如：[font=Consolas]Spawnertype[/font] 在 ESGUI 为刷怪笼处理插件时需要指定）
[/list]

[size=3][b]编辑物品：[/b][/size]
编辑物品与添加物品一样简单，只需要输入命令 [font=Consolas][b]/editshop edititem <选区> <序号> <键> <值>[/b][/font] 即可。

[list]
[*]你可能需要输入 [font=Consolas][b]/sreload[/b][/font] 才能让修改后的物品在商店中刷新。
[/list]
[b]命令分解：[/b]
[list]
[*][font=Consolas][b]/editshop edititem <选区> <序号> <操作> <键> <值>[/b][/font]
[*][font=Consolas][b]<选区>[/b][/font] = 所要编辑物品的选区
[*][font=Consolas][b]<序号>[/b][/font] = 所要编辑物品的序号。（正常情况下应当是商店 shops.yml 中的数字）
[*][font=Consolas][b]<操作>[/b][/font] = 键的操作。有效值为 [font=Consolas][b]add, set, remove[/b][/font]
[*][font=Consolas][b]<键>[/b][/font] = 你想要编辑的物品值，例如键“potiontypes”，“enchantments”，“spawnertype” 等...在此处可浏览所有的可用键名。
[*][font=Consolas][b]<值>[/b][/font] = 键的值，例如若你选择了“potiontypes”，你就需要在这个参数的位置输入一个有效的药水效果名称。
[/list]
[code]#命令格式在 yml 文件中的分解
#没有缩进
<商店/选区名称>: #在 config.yml 文件的商店或选区名称
  #2 个空格的缩进
  <index>: 商店中物品的序号
    #4 个空格的缩进
    <key>: "<value>" #应用至物品的键与值[/code]   
[size=3][b]删除物品：[/b][/size]
你应该也能猜到它有多简单，只需输入命令 [font=Consolas][b]/editshop deleteitem <选区> <序号>[/b][/font] 即可将指定物品从商店中删除。

[list]
[*]你可能需要输入 [font=Consolas][b]/sreload[/b][/font] 才能让删除的物品在商店中消失。
[/list]
[b]命令分解：[/b]
[list]
[*][font=Consolas][b]/editshop deleteitem <选区> <序号>[/b][/font]
[*][b]<选区>[/b] = 所要删除物品的选区
[*][b]<序号>[/b] = 所要删除物品的序号（正常情况下应当是商店 shops.yml 中的数字）
[/list]
[size=3][b]添加手持物品：[/b][/size]
也许你不想要手动添加物品然后辛苦编辑，所以你可以使用命令 [font=Consolas][b]/editshop addhanditem <选区> <购买价格> <出售价格>[/b][/font] 来添加已有物品至商店。

[list]
[*]在添加物品后你可能需要输入 [font=Consolas][b]/sreload[/b][/font] 才能让新添加的物品出现在商店中。
[/list]
[b]命令分解：[/b]
[list]
[*][font=Consolas][b]/editshop addhanditem <选区> <购买价格> <出售价格>[/b][/font]
[*][font=Consolas][b]<选区>[/b][/font] = 所要添加物品的选区
[*][font=Consolas][b]<购买价格>[/b][/font] = 设置玩家购买物品所需要花费的金钱
[*][font=Consolas][b]<出售价格>[/b][/font] = 设置玩家将物品出售至服务器可获得的金钱
[*]* 物品的[i][b]序号[/b][/i]在成功添加时会自动分配。
[*]* 命令会自动检查物品的内容，并在 shops.yml 下生成该物品的配置。
[/list]
[quote][color=Blue][b]！[/b][/color] 1.13+ 的 TAB 键补全对大部分命令的参数都很有用。[/quote]

[size=4][b]使用导入命令[/b][/size]

担心数据丢失？导入命令将会允许你导入下列内容：

[size=3][b]ShopGUI+：[i][color=Orange]*仅付费版[/color][/i][/b][/size]
本插件的付费版内置了一个从 ShopGUI+ 导入物品数据的命令。
[list]
[*]命令 [font=Consolas][b]/editshop import shopguiplus[/b][/font] 将会创建一个可被导入、快速且简单的物品/选区/设置摘要，这些内容不会直接导入到你当前的配置文件中。
[list]
[*]当摘要的结果符合你的要求是，你可以输入命令 [font=Consolas][b]/editshop import shopguiplus confirm[/b][/font] 来完成导入。
[/list][/list]
[size=3][b]EconomyShopGUI（免费版）：[i][color=Orange]*仅付费版[/color][/i][/b][/size]
若你刚开始使用本插件的付费版，且尚未改动过任何文件/商店；你可以使用导入命令将免费版的数据导入付费版的插件文件夹。

[list]
[*][font=Consolas][b]/editshop import EconomyShopGUI config.yml[/b][/font] 将会从免费版插件的文件夹导入 config.yml -+- 若你修改了配置文件中的任何设置，推荐你将它们一并导入付费版，以获得更流畅的体验。导入之后，请重启你的服务器，一些设置才可以生效。
[*][font=Consolas][b]/editshop import EconomyShopGUI shops.yml[/b][/font] 将会从免费版插件的文件夹导入 shops.yml -+- 非常推荐你将这个命令后于导入 sections.yml 的命令执行，以确保所有的商店正确载入。之后即可输入命令 [font=Consolas]/sreload[/font] 来应用改动。
[*][font=Consolas][b]/editshop import EconomyShopGUI sections.yml[/b][/font] 将会从免费版插件的文件夹导入 sections.yml -+-非常图鉴你将这个命令先于导入 shops.yml 的命令执行，以确保所有的商店正确载入。之后即可输入命令 [font=Consolas]/sreload[/font] 来应用改动。
[/list]
[size=3][b]Essentials 插件的 worth.yml[/b][/size]

[list]
[*][font=Consolas][b]/editshop import essentials worth.yml[/b][/font] 这将会将物品的售价从 EssentialsX 的 worth.yml 导入 EconomyShopGUI。需要注意的是在 shops.yml 中已有的物品售价会替换 worth.yml 中的售价。
[/list]
[quote][color=Blue][b]！[/b][/color] 当前的配置会复制至备份文件夹，用来在导入或升级插件时出现问题的情况下排除问题。[/quote]
[size=4][b]如何使用编辑商店命令来设置物品限购：[/b][/size]
物品限购为付费版独有功能，可在服务器上通过商店限购达到特殊物品的限量功能。

[list]
[*][font=Consolas][b]/editshop globalstock set <选区> <序号> <数量>[/b][/font] 该命令将会限制所有玩家在整个服务器中指定物品的购买数量为特定数值。
[*][font=Consolas][b]/editshop playerstock set <玩家> <选区> <序号> <数量>[/b][/font] 该命令将会限制指定玩家在整个服务器中指定物品的购买数量为特定数值。
[/list]

[quote][b][color=Red]！[/color][/b] 需要注意的是，没有在 shops.yml 中进行限购设置的物品不会让其成为限购物品，为设置物品限购，你需要先了解 shopss.yml 基础设置。[/quote]

[size=4][b]如何使用命令添加/修改物品：[/b][/size]
[quote][color=Blue][b]！[/b][/color] 若你想要添加一个键不在 additem 命令中的物品（例如：刷怪笼），你[i]必须[/i]编辑该物品并为其加上所需的键，否则物品将不会正常显示。

这些是你需要通过权限插件分配给指定权限组或玩家的基础权限。

1a. [font=Consolas]EconomyShopGUI.shop.all[/font] 允许打开 shops.yml 文件中定义的[i][b]所有[/b]商店[/i]
1b. [font=Consolas]EconomyShopGUI.shop.<选区>[/font] 允许打开 shops.yml 文件中定义的[i]指定商店（将 [font=Consolas]<选区>[/font] 替换为商店中选区的名称！）[/i]

[size=3][b]安装权限插件：[/b][/size]
安装权限插件非常简单，只需要下载后将 jar 文件放入服务器的 plugins 文件夹下即可。

[list=1]
[*]首先你需要找到你想要使用的权限插件。[url=https://www.spigotmc.org/resources/luckperms.28140/]我们推荐使用 LuckPerms[/url]，LP的 [font=Consolas]/lp editor[/font] 打开的网页编辑面板非常易于使用；
[*][b]熟悉你所使用的插件的基本操作步骤[/b]，确保它与你当前的版本[b]兼容且是最新版本[/b]；[[url=https://www.youtube.com/watch?v=Hfp5Sz3m5p4]简单的 LuckPerms 教程在这里可以查看[/url]]
[*]下载插件；
[*]将插件放入服务器的插件文件夹；
[*]重启服务器。
[/list][quote][color=Red][b]！[/b][/color]不要安装多个权限管理插件。它们之间的相互冲突会引起不必要的麻烦。:/[/quote]
[size=3][b]配置：[/b][/size]
我们不会为所有人详细解释如何设置权限。但是你只要在 Youtube 上搜索并[url=https://www.youtube.com/watch?v=Hfp5Sz3m5p4]观看过其他人关于这些的视频[/url]，你可以快速地按自己需求设置相应的权限！

在安装/配置好你的权限管理插件后，可在上方浏览 ESGUI 的完整权限列表。

[size=4][b]权限更新：[/b][/size]

[quote][b][color=Red]！[/color]更新插件会导致玩家无法使用 /sellall 和 /sellgui 命令出售物品，所以请阅读这一段文字！[/b][/quote]
在 4.4 版本后的 EconomyShopGUI 和 2.5 版本后的 EconomyShopGUI Premium，我们更新了全新系统，让服务器能更好控制玩家在商店中使用 /sellgui 和 /sellall 出售物品的行为。

玩家之前需要权限节点 [font=Consolas][b]EconomyShopGUI.shop.<选区>[/b][/font] 或 [font=Consolas][b]EconomyShopGUI.shop.all[/b][/font] 来通过 [font=Consolas][b]/sellall[/b][/font] 和[font=Consolas][b]/sellgui[/b][/font] 但这些在现在的版本按照命令分成了多个权限节点。

[size=3][b]怎么做？[/b][/size]

你基本上不需要担心玩家可以把什么物品卖到商店中，所以你只需要给予玩家下列权限即可：
[font=Consolas][b]➖ EconomyShopGUI.sellall.all[/b][/font]
[font=Consolas][b]➖ EconomyShopGUI.sellallitem.all[/b][/font]
[font=Consolas][b]➖ EconomyShopGUI.sellallhand.all[/b][/font]
[font=Consolas][b]➖ EconomyShopGUI.sellgui.all[/b][/font]
若你正在使用 LuckPerms（推荐）作为你的权限管理插件，你可以通过下列命令为你的权限组添加你所需要的上述权限：
[font=Consolas][b]➖ /lp group default permission set EconomyShopGUI.sellall.all true[/b][/font]
[font=Consolas][b]➖ /lp group default permission set EconomyShopGUI.sellallitem.all true[/b][/font]
[font=Consolas][b]➖ /lp group default permission set EconomyShopGUI.sellallhand.all true[/b][/font]
[font=Consolas][b]➖ /lp group default permission set EconomyShopGUI.sellgui.all true[/b][/font]
在另一方面，如果你有只能由特定玩家打开的商店在服务器中，你需要为他们单独添加打开这些商店的权限节点。

[size=5][b]排除问题：[/b][/size]
学会如何自行解决阻止插件启动的烦人配置和插件问题。

[size=4][b]01. 检查 [font=Consolas]latest.log[/font] 并寻找问题：[/b][/size]
插件将会记录问题、警告和服务器启动时的信息。先从这里看起！

[list]
[*]你的控制台日志有显示日志行数的限制，并不会在服务器启动完成后仍可浏览启动时的输出信息。它将会快速地在屏幕上划过然后消失；
[*]你可以在 <根文件夹>/[b]logs/[/b] 文件夹中找到 [font=Consolas]latest.log[/font]（以及被压缩打包的历史日志）；
[*]你应当能直接通过服务器提供商给予的“文件浏览器”功能访问服务器上的文件夹，或是通过 FTP、SFTP 等方式连接（若你找不到这些，你应当联系服务器提供商的客服）；
[*]如果你不能浏览服务器的文件，排查问题将会因为无法获取到日志文件而非常困难；
[*]你可以在 Discord 群组聊天上传或分享文件，具体方式如下：
[list]
[*][b]甲.[/b] 将文件后缀改为 Discord 服务器允许上传的扩展名（如 .txt 文件就能上传至服务器）
[*][b]乙.[/b] 复制日志文件的内容至诸如 https://hasteb.in/ 或 https://ghostbin.co/ 等的在线剪贴板网站（这些文件几周后将会自动删除），保存并复制这些文本的链接，并将这段网址粘贴在聊天栏中（也可以使用 https://pastebin.com/，但可能要求你设置过期时间）；
（译者注：推荐国内网友使用 https://paste.fastmirror.net/ 或 https://bytebin.lucko.me/ 等剪贴板网站。）
[/list][/list]

[size=3][b]02. 检查你是否使用了插件的最新版本：[/b][/size]

[list]
[*]在控制台或以管理员身份在游戏中输入命令 [font=Consolas][b]/version <插件名称>[/b][/font] - 这将会告诉你指定插件的版本，并显示是否有新版本可用。
[/list]
[size=3][b]03. 检查你的服务器是否是最新版本（Bukkit/Spigot/Paper）：[/b][/size]
[list]
[*]这些问题一般只会在[b]最新版本的 Minecraft[/b] 上出现。[i]旧版本的 MC 一般会有一个不再更新的最终构建放出[/i]；
[*]执行 [font=Consolas][b]/version[/b][/font] 命令将会显示服务器的版本，并显示你的构建版本落后最新版多少个版本；
[*]推荐在你使用需要 Minecraft 核心功能的时候将服务器更新到最新版本，有些情况下插件会使用当前构建版本并不存在的内容，并且我们也建议养成一个月更新一次服务端核心的习惯。
[*]注意：若你正在使用 Minecraft 最最新的版本（新到连 Mojang 都只是刚刚发布了正式版的那种）。你可以想象到会有许多的兼容性问题，这会持续几个月，除非开发者们更新了他们在最新版本的 Minecraft 的代码。总之请耐心。
[/list]
[size=3][b]04. 如何更新/安装插件（请参阅教程章节）[/b][/size]

[size=3][b]05. 请使用 /restart 命令，而不是 /reload 命令[/b][/size]

当插件还在运行的时候输入 /reload 命令来重载是非常不明智的：
[list]
[*]它不会正确更新你新安装的任何插件；
[*]它会破坏一些在开服初期需要初始化的插件，导致其重载后无法工作；
[*]即使 ESGUI 能够正常运转，其他插件很有可能在使用 /reload 命令之后损坏，因此强烈建议在添加/删除/更新插件后重启服务器。
[/list]

[*]这是一种检查插件是否被禁用的方式（在游戏内可见被禁用插件名称为红色），插件在启动时出现错误，因而会被禁用，这时候你就需要找到 [b]latest.log[/b] 来具体排查其中的原因。

[size=3][b]10. 识别冲突的插件和命令：[/b][/size]
[b]A. 插件列表：[/b]
[list]
[*]在服务器上[b]你只能拥有一个权限插件[/b]。如果安装多于一个的权限管理插件可能会造成奇怪的权限问题；
[*]你选择的大部分权限插件都可以正常运转，但请只使用一个，并确保它会和你的服务器版本兼容；
[*]通用的权限插件：[url=https://www.spigotmc.org/resources/luckperms.28140/]LuckPerms[/url]（强烈推荐！）、PowerRanks、GroupManager、PermissionX（即 PEX。已经停更，慎用）、MyPermissions、bPermissions、zPermissions、PowerfulPerms、PermissionsBukkit、BungeePerms 等...
[*]Bukkit 的 permissions.yml 不受本插件支持/不会正常工作。请尽量使用权限插件。
[/list]
[b]B. 命令：[/b]
[list]
[*]请先阅读你所有插件的页面介绍/维基、命令列表、权限列表和更新日志。了解插件功能会让你在配置插件时更有余地，学会配置是作为一个优秀的服主/管理员/维护者所必需的；
[*]如果两个插件注册了同一个命令，例如 [font=Consolas]/shop[/font]，你可以：
[list]
[*][b]A. [/b]移除不需要却注册了相同命令的插件；
[*][b]B. [/b]使用 commands.yml（在服务器的根目录）来覆盖或重设命令优先处理的插件。
[/list][/list]

[b]上述情况：[/b]
[list]
[*]如果你存在着冲突命令、插件、权限或其他内容，你可以逐个找出造成问题的插件，并将其禁用或卸载，一些简单的规则可用找到问题插件：
[list=1]
[*]出现问题时你是否添加/修改/删除了什么？
[*]找到可能存在问题的插件，将其删除；
[*]重启服务器；
[*]测试/复现问题。
[list]
[*]若问题存在，则按照第一步重新开始；
[*]若问题解决，那么最近一次禁用或删除的插件可能是问题的根源 - 试着浏览插件文档来解决问题！
[/list][/list][/list]
[quote][b]若你有一个测试服务器或一个插件管理插件（例如 Plugman）：[/b]你可以不用重启而使用插件的管理或卸载功能来复现或解决插件的问题。在游戏内载入/卸载插件很有可能不受支持，并有可能在正确重启之前不能正常工作。[/quote]

[b]Bukkit 的 [font=Consolas]Commands.yml[/font] 文件示例（也可见操作教程的 YAML 格式）：[/b]
[code]aliases:
  promote:
  - lp user $1 promote ranktrack
  # 上述命令会创建一个别名命令，在玩家执行的时候将执行下方的命令，并将参数位置替换为玩家的名称
  demote:
  - lp user $1 demote ranktrack
  # 与上述相同，但是是降级命令
  superpickaxe: []
  # 上述设置将禁用 superpickaxe 命令
  dsrv:
  - discord $1-
  # 上述设置为 discordSRV 插件创建了一个别名命令 (命令的等价用法)，该命令将会将其之后的所有参数都传递至此处
  shop:
  - EconomyShopGUI:shop $1-
  # 上述设置将会使 /shop 命令由 ESGUI 处理 (并接受任意数量的参数等内容)
  shop2:
  - 其他插件:shop $1-
  # 上述命令将会打开在执行 /shop2 命令时交由“其他插件”处理 (并接受任意数量的参数等内容)[/code]

[size=3][b]11. 如何重置插件内容[/b][/size]
我们已经试过了很多东西，我们还是需要重新开始...

[list=1]
[*]Jar 文件可能是损坏的（可见操作教程 04 - 如何升级/安装插件）
[list]
[*]从插件的资源页下载插件（如：[url=https://www.spigotmc.org/resources/economyshopgui.69927/]Spigot[/url]）；
[*]从服务器目录 [font=Consolas]plugins/[/font] 下[i]删除[/i]原有的插件 [i]jar[/i] 文件（建议先重启服务器）；
[*]将你刚才下载的文件放入服务器下的 [font=Consolas]plugins/[/font] 目录；
[*]重启服务器。
[/list]
[*]若有需要，重置插件的配置文件：
[list]
[*]按需备份已有的插件配置；
[*]删除所有 [font=Consolas]plugins/EconomyShopGUI[/font] 文件夹下的内容，或将它们重命名
[list]
[*]这是因为插件只会检查 [font=Consolas]sections.yml[/font]、[font=Consolas]shops.yml[/font]、[font=Consolas]config.yml[/font] 文件和 [font=Consolas]LanguageFiles[/font] 文件夹，而忽略其他文件的存在。
[/list][/list][/list]

[size=4][b]更新/安装插件[/b][/size]

当插件有更新可下载时，在插件出现重大更新等需要改动或升级配置文件的情况时，浏览更新日志很重要（这些内容通常会出现在插件资源页面）。

[list]
[*]更新前记得备份！我们越不希望文件丢失或被覆盖，这种意外就越有可能发生。
[/list]

当你阅读了更新日志之后，是时候更新插件的 jar 文件了，它很简单。你只需要替换服务器根目录的 plugins 文件夹中旧的 jar 文件即可。

[list]
[*]最好[b]删除原有的 jar 文件[/b]，然后手动[b]上传新的文件[/b]来避免文件损坏的隐患。
[/list]
[quote][b][color=Blue]！[/color][/b]大多数的插件名称后会带有版本号。例如：[font=Consolas]EconomyShopGUI 3.0.8[/font][/quote]

[size=4][b]如何找到你的插件文件[/b][/size]

[quote][b][color=Red]！[/color][/b]编辑文件前请先了解有关 YAML 的语法知识！该格式的配置不容忍任何错误！请在保存前[url=https://jsonformatter.org/yaml-validator]验证语法有效性[/url]！[/quote]

[list]
[*]首先你需要找到服务器的 [font=Consolas]plugins[/font] 文件夹（[b]请联系你的服务器提供商获取打开服务器文件夹的方法[/b]）
[*]在 [font=Consolas]plugins[/font] 文件夹中，找到 [font=Consolas]EconomyShopGUI[/font] 文件夹并打开（若你购买了付费版，则你需要找到 [font=Consolas]EconomyShopGUI-Premium[/font] 文件夹）
[*]在这里你会找到就的 [font=Consolas]config.yml[/font]、[font=Consolas]Example Items.txt[/font]、[font=Consolas]shops.yml[/font] 和 [font=Consolas]sections.yml[/font]（部分设备中的后缀名称可能是隐藏的）
[*]用服务器的面板管理系统或 [url=https://notepad-plus-plus.org/downloads/]Notepad++[/url] 打开文件。（这些是最简单的办法，你也可以使用 FTP/文件同步系统，虽然会略微复杂一些。请联系服务器提供商以获取更多信息）
[*]现在可以开始编辑你的配置文件/商店/选区！请见 config.yml、sections.yml、shops.yml 和 语言文件章节来获取更多细节。
[*]需要注意的是，YAML 不能包含任何语法错误。所以在联系获取帮助之前请[url=https://jsonformatter.org/yaml-validator]确保你的文件没有基础的语法错误[/url]。
[/list]

[size=4][b]升级商店和选区配置而不丢失任何已有改动（备份文件）[/b][/size]

若要更新这些配置，你先要找到插件的 plugins 文件夹。

若要确保你没有丢失在商店和选区内的任何数据/改动，你可以：[b]备份文件并将其转移到其他地方保存，同时删除原本的文件。或是将文件从 [font=Consolas]shops.yml[/font] 重命名为 [font=Consolas]shops-old.yml[/font]，从[font=Consolas]sections.yml[/font] 重命名为 [font=Consolas]sections-old.yml[/font][/b]。（任何名称均可，但绝不能是原本的名称，因为这会被插件识别并导致覆盖。）

若插件找不到商店、选区、语言或配置文件，它会在服务器重启或你输入 [font=Consolas]/sreload[/font] 尝试重新生成一份默认配置（选项、值、价格、显示名称等等...）

现在你有两个商店和选区的配置（新版和旧版）。如果你的操作正确，当前的配置文本名称应当为 [font=Consolas]shops.yml[/font] 和 [font=Consolas]sections.yml[/font]，并且均位于 EconomyShopGUI 文件夹下。

最重要的异步，你需要复制旧配置文本中所有的数据/改动到新文件中，但注意要保留新出现的选项不被修改（例如选项名称、功能...），不然的话就可能会让插件无法使用。

[list]
[*]在 ESGUI 更新后，一些键和其他设置的格式会产生变化，选项 [font=Consolas]config-version: #[/font] 的数字也会产生变化。
[*]若你找到了一些不能正确工作的内容，请先检查文件中的键/设置是否与该维基的最新版本和最近的更新日志相同！
[/list]
如果你操作正确，重启/重载服务器或插件之后，游戏内的商店将会使用你之前所做的修改配置来显示内容。

若你在后台找到了有关的报错（[url=https://pastebin.com/DNF5frGr]例如这个[/url]）或插件被禁用。很有可能是因为新的配置文件中出现了错误。浏览排除问题章节来获取有关解决方法。

[size=4][b]如何存储/分享物品形式的经济单位：[/b][/size]

（将经济存储在你的 Vault/经济插件 或 [font=Consolas]/pay[/font] 命令的替代方法）

设置一个物品的附魔，或其他方法使其在生存中不可获取，并将其的买价和卖价设置为相同（使用隐藏的 NBT 标签会更为方便，但这是付费版功能）。示例：[code]  `1000`:
    material: PAPER
    name: "&a$5000 兑换券"
    buy: 5000
    sell: 5000
    lore:
    - "&a$5000 兑换券"
    - "可在商店处兑换并存入账户"
    NBTData: {ESGUI:{Voucher:5000}} #付费版独有功能[/code]
注意：该功能与现实中从银行存取钱是相同的。若你不小心遗失，那么这部分钱将永远不能找回。取出/存入的钱在任何支持“银行/基于 Vault 的经济插件”中都可使用（不只是 ESGUI）。

[size=4][b]如何创建基于物品的经济：[color=Orange]*付费版独有功能[/font][/b][/size]

物品经济已经是本插件的独特功能，但它在你的游戏能创建自定义物品货币时能更加有用。

[size=3][b]操作步骤：[/b][/size]

EconomyShopGUI 如果没有简单的步骤来实现这个功能，那么它就不会有着操作简单的卖点。

若要为一个物品设置自定义货币，你需要输入下列命令：

[list]
[*][b]选区[/b]
[list]
[*][b]命令：[/b][font=Consolas]/editshop editsection <选区> set economy handItem[/font]
[/list][*][b]商店物品[/b]
[list]
[*][b]命令：[/b][font=Consolas]/editshop edititem <选区> <物品> set economy handItem[/font]
[/list][/list]

这些命令将会将你手上的物品设置为可被识别的物品货币，并将其应用到指定的选区/物品。免去你自己设定格式的繁琐步骤！

额外地，你可以从配置文本中复制配置的格式，放入默认的经济提供插件中。

[size=3][b]格式：[/b][/size]

[b]示例：[/b]
[list]
[*]以玩家头颅实现的自定义物品经济：
[list]
[*][b]格式：[/b][font=Consolas]economy: ITEM:material:PLAYER_HEAD name:&6&lCustom coin skullowner:Notch id:Creator[/font]
[/list][*]以带有自定义名称和描述及自定义模型数据（Custom Model Data）实现的自定义物品经济：
[list]
[*][b]格式：[/b][font=Consolas]economy: ITEM:material:ARROW name:&6&l狙击枪弹药 lore:&c该子弹可用于下列枪械的装填:\n&a巴雷特\n&aP90 CustomModelData:1003 id:Bullets[/font]
[/list][*]以带有自定义附魔和 NBT 数据实现的自定义物品经济：
[list]
[*][b]格式：[/b][font=Consolas]ITEM:material:Diamond name:&9Real money enchantments:SHARPNESS:5,EFFICIENCY:3,MENDING,UNBREAKING:2 NBTData:{Damage:5}[/font]
[/list][/list]

[size=3][b]自定义货币显示名称：[/b][/size]

使用货币的 ID，你可以设置物品的显示名称，这是用于展示在商店物品上的货币名称。

你要做的第一步就是通过“id”标签给予物品一个 ID，见上述内容可了解它要如何完成。

之后你就可以在 config.yml 下通过 ID 设置自定义显示名称，在这里可以了解如何制作自定义物品的货币经济。

[b]允许的物品选项：[/b]
[list]
[*][b]id[/b]
[list]
[*][b]描述：[/b][i]用于作为经济物品的 ID，这一般用于货币的单数和复数显示名称。[/i]
[*][b]示例：[/b][font=Consolas]id:Bullets[/font]

[b]1. 使用命令 [font=Consolas]/npc create Test[/font] 创建一个 NPC，见[url=https://wiki.citizensnpcs.co/Commands]这个维基[/url]来获取命令的更多用法。[/b]

[size=4][b]如何设置 DiscordSRV 联动：[/b][/size]

DiscordSRV 联动允许插件记录通过 GUI 界面和命令产生的交易记录，并将其发送在 Discord 的指定频道中。见下文来为你的服务器设置这样的交易同步操作。

[list=1]
[*]安装并配置 DiscordSRV 插件，确保其正常运行。你可以在他们的网站上找到[url=https://docs.discordsrv.com/Installation/]相关教程[/url]；
[*]当 DiscordSRV 启用后，你需要先在该插件的 config.yml 下建立一个用于发送日志的频道。见[url=https://docs.discordsrv.com/installation/initial-setup/#set-up-a-basic-channel-relay]此[/url]获取建立频道的有关信息；
[*]在创建并启用 DiscordSRV 的频道后，使用命令 [font=Consolas]/discordsrv reload[/font] 来重载插件，并在 EconomyShopGUI 的 config.yml 下，设置 [font=Consolas]enable-discordsrv-hook: true[/font]；
[*]将频道[b]名称（不是 ID）[/b]添加至 DiscordSRV 联动设置下；
[*]按喜好配置将要发送至 Discord 聊天群组的聊天信息，默认情况下它会使用嵌入样式的消息，但你也可以通过添加 [font=Consolas]raw: ""[/font] 改为发送原始消息；
[*]最终你需要通过命令 [font=Consolas]/srelaod[/font] 重载 EconomyShopGUI，当正确配置上述内容时，你将会见到 Discord 聊天群组中的交易信息开始出现！

[size=5][b]限制物品[/b][/size]
本页面介绍了付费版的 EconomyShopGUI 限制物品功能

[quote][b][color=Green]√[/color][/b]你可以对同一个物品使用出售限制和库存限制。[/quote]

[size=4][b]限制库存[/b][/size]

限制库存功能在你想要限制某些特殊物品时很有用，这能让它只能在小范围内流通。

这个功能有两种限制模式：

[list=1]
[*][b]全局购买限制[/b] = 限制全服（每个玩家）只能购买一定数量的指定物品。
[*][b]玩家购买限制[/b] = 按玩家限制物品的购买量。
[/list]

你也可以指定一个补货时间，在物品售罄时会开始倒计时，在结束后为玩家或全服补货。

该功能的相关数据会存储在一个 SQLite 数据库中，使得服务器重启之后的补货倒计时能够继续正常工作。

[quote][b][color=Green]√[/color][/b]当使用物品选项“refill-stock”，物品的玩家或全局补货数量会在物品重新卖给商店时增加。[/quote]

[b]全局库存：[/b]

对物品使用设置 [b][font=Consolas]stock-limit-global[/font][/b] 可指定物品的库存数量。

所有玩家共享剩余可购买的物品数量，若某个玩家购买了 5 个物品，那么其他玩家能够购买的数量也会相应地减少 5 个。

当其他玩家购买了一定数量的物品，使其的库存下降到 0，也就是没有足够的库存可供出售时，其他尝试交易的玩家将收到一条提醒。告知他们商品已被售罄。

对物品使用设置 [b][font=Consolas]auto-restock-global[/font][/b] 将会在倒计时为 0 时自动重置库存。所用时间长度由指定时间单位和具体数量构成。例如将该项设置为 [font=Consolas][b]auto-restock-global: 14d 12h 10m[/b][/font] 则会在售罄 14天12时10分 后补货。

[size=3][b]玩家库存：[/b][/size]

对物品使用设置 [b][font=Consolas]stock-limit-player[/font][/b] 可分玩家指定物品的库存数量。

玩家一开始都可购买一定数量（预先设置）的物品，与上述设置不同的是，限制购买的数量按玩家分别计算。

所以例如你将这个设置的值调整为 5，每个玩家都可以购买最多 5 个这样的物品。这也对命令商品有效。

对物品使用设置 [b][font=Consolas]auto-restock-player[/font][/b] 可以让玩家在物品被买空的时候补货。不同玩家之间不会互相影响，例如玩家 A 的补货不会将玩家 B 的库存一并补满。这条设置的倒计时格式与上述全局补货是相同的。

[size=4][b]出售限制[/b][/size]

使用出售限制的物品，就像你想的那样，只能出售固定的数量。这个功能可以和限量出售搭配使用。

限制出售的方式可以按玩家，也可以按全体：

[list=1]
[*][b]全局出售限制[/b] = 整个服务器的玩家只能向商店出售一定数量的物品。
[*][b]玩家出售限制[/b] = 服务器中的每个玩家都能向服务器出售一定数量的物品。
[/list]
这允许你指定物品可再（由商店）收购的清空库存时间。

[size=3][b]全局出售限制：[/b][/size]

使用全局出售限制将能设置卖给商店的物品数量上限。

全局出售限制能是的玩家只能在服务器中向商店出售一定数量的物品。例如，玩家向商店出售了 10 个物品，且全局出售限制数量被设定为了 10，那么其他玩家就不能再向商店出售同样的物品。

对物品使用设置 [b][font=Consolas]auto-restock-global-sell[/font][/b]，则商店可在一段时间后清空该物品的库存，使玩家可以再次向商店出售物品。与补货相似，这个设置使用的时间格式是相同的，例如 [font=Consolas][b]30m 45s[/b][/font] 表示售满之后等待 30 分钟又 45 秒 即可重新向商店出售。

[size=3][b]玩家出售限制：[/b][/size]

对物品使用选项 [b][font=Consolas]sell-limit-player[/font][/b] 可以指定按玩家的出售限制。

例如将该设置的值设置为 64，那么每个玩家都可以向商店出售 64 个这样的物品，且剩余可出售的数量不会相互影响。

对物品使用选项 [b][font=Consolas]auto-restock-player-sell[/font][/b] 则会在商店剩余空间不足时的一段时间后清空库存，使得玩家可以再次向商店出售这些物品。

[size=4][b]补货数量/限制：[/b][/size]

目前，在商店售罄时有两个补货选项：

[size=3][b]定时补货：[/b][/size]

物品能够在一段时间后自补货/重置限制。

需要记住的是，补货或库存清空只会在玩家完全售空/售满时触发倒计时。

所以，一个例子是：
如果有一个限购 50 个的物品，且该物品设置了 [font=Consolas][b]auto-restock-global/player: 10m[/b][/font]，那么玩家在售空/售满商店后等待 10 分钟才可再次进行购买或出售操作。

[size=3][b]固定频率：[/b][/size]
[quote][b][color=Blue]！[/color][/b]这部分的内容尚在制作且不可用，敬请期待！[/quote]
使用固定时间将会允许你在指定时间补货，例如每个小时的第 25 分钟或星期天的上午七点。

和补货时间设置不相同的是，这个频率会强制执行补货操作，[b]即使[/b]玩家在这些时间点并未售空/售满商店的物品。

与其他 cron 表达式的不同的是，我们使用了流行的 java Quartz Cron Scheduler，这会和普通的 cron 表达式有略微的差别。下列文本是 cron 的有效表达格式：[font=Consolas][b]auto-restock-player-frequency: 0 0 0 ? * *[/b][/font]。

有效的定时执行格式有 6 或 7 个值，它们分别对应：秒、分、时、月份中的日期序号、月、星期的天数和可选的年份。
见下表来获取更多信息。
[img]https://2522957397-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2F-M6QXvrDDroId1RhlXgl%2Fuploads%2FoRAd1o3j2e3fkKrG3qvf%2Fimage.png?alt=media&token=5ca29350-739f-4bed-9d0d-5a191d3ee300[/img]

[size=5][b]有关 YAML（YML 文件）[/b][/size]
你弄坏了它，就要修好它。

[quote][b][color=Blue]！[/color][/b]YAML 有着非常具体的语法，且不能出现任何不符合格式的缩进/符号！[url=https://jsonformatter.org/yaml-validator]使用之前请验证你的文件是否符合格式！[/url][/quote]

在编辑你的 YML 文件时，请先确定在非英文双引号包括的区域外[b]严格遵守[/b] YAML 的语法。

[size=4][b]YAML 格式：[/b][/size]

[list]
[*]英文冒号 [font=Consolas]:[/font] 和值之间必须有一个空格；
[*]每个文件中的“缩进”均为两个空格（不可以使用 TAB 制表符）；
[*]以英文井号 [font=Consolas]#[/font] 开头的为注释，这之后的文本会被插件无视；
[*]若在修改文件后遇到错误，或者你尚未熟悉编辑这些文件的方法。[url=https://jsonformatter.org/yaml-validator]请善用在线 YAML 格式检查器[/url]。它会帮你找出文件中任何[b]你需要[/b]在实际使用前[b]修复[/b]的格式错误。
[/list]

[b]常见错误：[/b]

[list]
[*]缩进不正确（不符合缩进空格数）；
[*]缺少空格）：英文冒号和值之间的空格缺失；
[list]
[*]例如（displayname:"<在英文冒号和英文双引号之间的空格缺失"）
[/list][*]缺少“内容结束/起始符”：你忘记了在某些地方补上英文双引号；
[*]绝对不要加入任何 TAB 制表符。
（译者注：部分编辑器可自动将制表符转化为空格。）
[/list]
[size=4][b]配置阐释：[/b][/size]

[size=3][b]Shops.yml：[/b][/size]

[b]shops.yml 的缩进：[/b]

[list]
[*]商店选区标题缩进为 0（零）个空格；
[*]物品数字/序号缩进为 2（二）个空格；
[*]物品的特定设置/值缩进为 4（四）个空格；
[*]浏览 Shops.yml 章节来获得更多信息。
[/list]
[code]Armor: #商店选区名称前没有空格缩进, 末尾必须有英文冒号“:”. 名称必须与 sections.yml 下设置的相同。
  '1': #在 <序号/物品数字> 下有两个空格缩进, 末尾必须有英文冒号“:”.
    material: GRASS_BLOCK #这些设置需要四个空格缩进, 末尾必须有英文冒号“:”和一个空格, 然后是你要设置的值
    buy: 20
    sell: -1

    buy: 10
    sell: 2

    buy: 20
    sell: 2

[size=4][b]“完整”技术阐释：（既复杂，又简单）[/b][/size]
[code]#商店文件示例:
# YAML 文件中的第一级内容不应包含缩进空格
####选区名称不可以包括任何的空格，但可以包含下划线“_”
####选区名称应当与你在 sections.yml 为其设置的名称完全匹配.
Armor: #英文冒号“:”后不应有任何内容. 它是下文详细设置的开头。
  #第二级应当包含两个缩进. 这些是上方定义值 (Armor) 的详细信息
  ####(仅 shops.yml) 注意: ESGUI 不要求下列内容为数字. 你可以使用词语或大部分字符. 
  ####这是会在 /shopedit 命令中使用的 <序号/物品数字>.
  ####1.12 以下的版本建议使用数字序号, 
  ####1.13+ 拥有 TAB 补全功能，可帮助玩家在输入参数时更快选择，为物品添加一个详细的命名对命令补全过程更有帮助。

  '1': #再次，英文冒号“:”后不应有任何内容，因为它是下文详细设置的开头...
  #
    #在序号/物品下的是更高一级的缩进, 并且需要两个额外的空格缩进. 
    #也就是说这几行需要四个空格缩进.
    material: GRASS_BLOCK 
      #英文冒号“:”后必须有相应的值，值和冒号之间必须有一个空格.
      #在给定值包含特殊符号的时候才需要将其用英文双引号包括
    #仍需要四个空格缩进，因为这些也是 '1' 下的设置的之一      
    buy: 20
    #### 将对应选项修改为负数即可禁用操作 '物品不能购买/出售'
    sell: -1
    lore: #下面几行又是一个更高级的缩进, 但这个列表中的每一行都需要以“-”开头
    #在这种情况下我们需要的是文本, 所以我们应当将“-”后的内容以 英文双引号“""”括起
    #若你需要在你的文本中使用英文双引号，你需要将它输入两次来“转义”. (必须以英文双引号开头和结尾)
    ####Minecraft 的聊天颜色字符也同样支持
    - "示例描述"
    - "&c描述的 ""第二"" 行"
    enchantments:
    #另一种列表格式为数字. 在 ESGUI 中一般用于定义附魔及其等级. 
    #它的格式为附魔名称开头，以英文冒号“:”连缀，其后紧跟附魔等级
    - EFFICIENCY:5 
    - FORTUNE:3[/code]

[size=5][b]Sections.yml[/b][/size]
你将会逐步学习选区配置的每个选项，并了解如何正确配置它们。

[b][size=4]选区配置的总体信息[/size][/b]

这个配置文件存储着所有用于创建商店选区的自定义数据，会在你输入 [font=Consolas]/shop[/font] 命令后出现在主菜单。
你可以在这里指定物品的显示名称、设置物品的样式、添加或插入物品描述并设置商店选区的图标在主商店中的展示位置。
[quote][b][color=Red]！[/color][/b]编辑文件前请先了解有关 YAML 的语法知识！该格式的配置不容忍任何错误！请在保存前[url=https://jsonformatter.org/yaml-validator]验证语法有效性[/url]！[/quote]

[b]配置文件结构：[/b]
[code]###### 配置格式参考: ######
ShopSections:
  <section>:
    <option>: <value> #单值
    <option>: #列表格式必须支持多个值. (描述, 附魔等..)
    - <value1>
    - <value2>
###### 示例: ######
ShopSections:
  Blocks: #这个名称必须与 shops.yml 中的名称完全匹配! 不可以有任何空格!

    enable: true
    material: "GRASS_BLOCK"
    displayname: "方块"
    slot: 19
    lore:
    - "Basic building blocks"[/code]
    
[size=4][b]你可使用的所有选项及其释义[/b][/size]

[size=3][b]物品选项列表：[/b][/size]

[list]
[*]这些设置的功能与 shops.yml 中的相同，但是会有一些 sections.yml 中的设置存在着细微的差距：

[*][b]enable:[/b] - 完全启用或禁用图标的[b]可见性和[/b]进入商店的[b]功能[/b]
[*][b]material:[/b][i][必需][/i] - 商店选区图标在主商店中的样式
[*][b]displayname:[/b] - 商店选区的显示名称，会在选区未指定标题时使用这条内容。（可使用彩色字符）
[*][b]slot:[/b][i][必需][/i] - 指定商店选区图标在主商店页面的位置（必须为 0 - 44 的数字，额外页数尚不支持，不能和其他物品重复）
[*][b]title:[/b] - 商店内的标题。请不要将它与显示名称搞混。
[*][b]lore:[/b] - 商店内物品末尾的统一描述（主商店中选区物品的描述，支持彩色字符）
[*][b]potion-glow:[/b] - 指定药水的药水颜色。（仅在物品材料为药水、喷溅型药水、滞留型药水、药箭时有效）
[*][b]enchantment-glint:[/b] - 指定物品是否有附魔光效。
[*][b]skullowner:[/b] - 为头颅赋予自定义的拥有者（仅在该物品为玩家头颅时有效）
[*][b]armorcolor:[/b] - 指定皮革盔甲的 hex 颜色值（仅在该物品为任意种类的皮革盔甲时有效）
[*][b]hidden:[/b] - 该商店选区是否在主菜单上可见并可通过主菜单选择进入
[*][b]subScetion:[/b] - 是否将该选区设置为子选区
[*][b]full-item:[/b] - 将物品设置为仅供展示用的物品。
[*][b]economy:[/b] - 设置该选区使用的默认经济种类。
[*][b]NBTData:[/b] - 指定物品的 NBT 数据 [i][b][color=Orange]*仅付费版[/color][/b][/i]
[*][b]skull-texture:[/b] - 赋予头颅指定皮肤 [i][b][color=Orange]*仅付费版[/color][/b][/i]
[*][b]dynamic-pricing:[/b] - 是否启用动态定价功能 [i][b][color=Orange]*仅付费版[/color][/b][/i]
[*][b]click-commands:[/b] - 进入/点击商店选区后执行的命令列表 [i][b][color=Orange]*仅付费版[/color][/b][/i]
[/list]

[size=3][b]主商店界面填充物品：[/b][/size]

主商店填充物品用于填充主商店的空白格子。
修改该物品的选项位于 sections.yml 配置的最下方。

[font=Consolas]material[/font] 为物品的材料。例如，将 [font=Consolas]"GRAY_STAINED_GLASS_PANE"[/font] 替换为 [font=Consolas]"RED_CARPET"[/font]，可使菜单中的填充物替换为红地毯。

若要启用显示名称选项，你需要将 [font=Consolas]displayname[/font] 项前的注释符号去除，名称可按你需求随意设置，且支持[url=https://www.digminecraft.com/lists/color_list_pc.php]彩色字符[/url]。

[code]#主商店中填充空白格子所用的内容
Fill-item:

  material: "GRAY_STAINED_GLASS_PANE"
# displayname: "&8此地无银三百两"
  # 将该值设置为 false 来取消该选项
  enable: true[/code]

[b][size=4]商店配置的全局信息[/size][/b]

这份配置文件存储着所有商店中的物品，且按选区各自存放。
可按你喜好任意编辑物品！
你可以添加任意数量的物品至商店，插件会自动计算容纳商品所需的页数！

[list]
[*]来自其他插件的自定义物品仅付费版 ESGUI 支持（除非它们是免费版的配置文件制作而成的物品）
[/list]
[quote][color=Blue][b][font=Consolas]i[/font][/b][/color]我们推荐你将本插件中售卖的物品按服务器经济状况适当更改。服务器种类有许多，如空岛、生存、派系、PVP等，大多数服务器需要按服务器状况修改商店的内容，并重新设置商店中物品的价格，若你要原封不动地使用本插件提供的默认配置，也不是不可以。你也可以浏览我们在 Discord 聊天群组的频道 #Shop-Files-Sharing 来寻找适合你的商店配置。[/quote]

[list]

[size=3][b]物品材料（Material）[/b][/size]

[list]
[*]此为商店中列出/待售的物品或方块的 ID。
[*]它必须为符合你的版本的[url=https://www.digminecraft.com/lists/item_id_list_pc.php]有效物品名称[/url]！
[*]材料需为有效的物品 ID。你可以从下面的链接得到相关的内容：
[list]
[*][url=https://www.digminecraft.com/lists/item_id_list_pc.php]1.16 或更低的版本[/url]
[*][url=https://minecraftitemids.com/]1.18 的物品[/url]
[*][url=https://docs.google.com/spreadsheets/d/1-Jf8VpmG-V2ytZ0ppAEDtVSaVYEjQ-CYAZMt-eQMNxQ/edit#gid=984941200]1.19 的物品，由 CommandLeo 制作[/url]
[/list][*]ID 的“命名空间”为代码赋予物品的名称
[*]如：[font=Consolas]experience_bottle[/font]
[*]此 ID 不会对大小写敏感。
[/list]

[quote][b][font=Consolas][color=Blue]i[/color][/font][/b]出于兼容性原因：在 1.13 “扁平化”更新后的物品名称在 1.8-1.12 的版本仍然可用。[/quote]
[quote][b][color=Green]√[/color][/b] 你可以通过将材料编辑为 AIR 创建一个虚拟占位物品。[/quote]

[b][size=3]购买价格（Buy）：[/size][/b]

[list]
[*]此为玩家购买物品所需支付的价格
[*]可以为任何数
[*]这里设置的数字是交易后从玩家账户中扣除的数量（存储在你兼容 Vault 的经济插件中）
[*]这个值只会在商店中显示，不会在任何购得的物品上显示
[/list]
[quote][color=Blue][b][font=Consolas]i[/font][/b][/color]将该选项的值设置为负数可使得物品不能购买。[/quote]

[size=3][b]出售价格（Sell）：[/b][/size]

[list]
[*]此为出售给服务器时物品的价值。
[*]可以为任何数
[*]此为玩家出售物品给服务器时可获得的金钱数量（所获金额直接汇入玩家账户）
[*]这个值只会在商店中显示，不会在任何购得的物品上显示
[/list]
[quote][color=Blue][b][font=Consolas]i[/font][/b][/color]将该选项的值设置为负数可使得物品不能出售给服务器。[/quote]

[size=3][b]物品名称（Name）：[/b][/size]

[list]
[*]这会为物品指定一个名称，会显示在玩家购买后收到的物品上。在 [font=Consolas]displayname[/font] 项未设置的时候也会在商店中显示。
[*]可以包含[url=https://www.digminecraft.com/lists/color_list_pc.php]彩色字符[/url][url=https://www.digminecraft.com/lists/color_list_pc.php]（颜色/格式字符）[/url]。
[/list]

[*]可使用内建变量 [font=Consolas]%buyPrice%/%sellPrice%[/font] 表示购买/出售价格。
[/list]

[size=3][b]展示描述（Displaylore）：[/b][/size]

[list]
[*]请不要将它与选项 [font=Consolas]lore[/font] 弄混，因为这个选项只会影响商店展示物品的描述（鼠标悬浮在物品上显示的内容）
[*]为物品添加自定义的描述！
[*]它可以包含[url=https://www.digminecraft.com/lists/color_list_pc.php]彩色字符[/url][url=https://www.digminecraft.com/lists/color_list_pc.php]（颜色/格式字符）[/url]。
[*][*]可使用内建变量 [font=Consolas]%buyPrice%/%sellPrice%[/font] 表示购买/出售价格。
[/list]
[code]Displaylore_item_shop:

    sell: 5

    displaylore: # 商店中显示的物品描述
    - '&1Gypopo'[/code]

[size=3][b]隐藏价格描述（HidePricingLore）：[/b][/size]

[list]
[*]选项 [font=Consolas][b]hidePricingLore[/b][/font] 顾名思义，会隐藏默认的定价描述，以便你添加自己的价格描述。
[*]这也会在物品能购买的情况下隐藏价格描述
[*]在制作[i]装饰性/占位[/i]物品时很有用
[/list]
[code]HiddenPricingLore_item_shop:

  1:
    material: LAVA_BUCKET
    buy: 200
    sell: 75
    hidePricingLore: true # 这会对玩家隐藏价格
  
  2:
    material: WATER_BUCKET
    buy: -1
    sell: -1
    hidePricingLore: true # 这会隐藏默认的 '物品不可交易' 描述[/code]
[quote][b][color=Green]√[/color][/b] 该选项可禁用单个物品的价格显示，但你也可以清空语言文件中这些物品的文本，这样每个物品都不会包含价格描述。[/quote]

[size=3][b]物品槽位（Slot）：[/b][/size]

[list]
[*]选项 [font=Consolas][b]slot[/b][/font] 可决定物品在商店中的位置
[*]例如，设置 [font=Consolas][b]slot: 14[/b][/font] 表示它在商店的第一页的第 14 格
[*]也可为“slot: 55”，这样物品就会在第二页的第 10 格出现。
[/list]
[code]
Slot_item_shop:

  1:
    material: chainmail_chestplate
    buy: 25
    sell: 10
    name: "&d&l骑士装备"
    slot: 11

  2:
    material: iron_chestplate
    buy: 35
    sell: 15
    name: "&d&l骑士装备"
    slot: 13

  3:
    material: diamond_chestplate
    buy: 50
    name: "&d&l骑士装备"
    slot: 15
    
  4:
    material: netherite_chestplate
    buy: 100
    name: "&d&l骑士装备"
    slot: 17[/code]

[quote][b][color=Green]√[/color][/b] 物品的格子位置决定了物品在整个商店选区中的位置，不只是商店页数！例如 slot: 445 会将格子移动到商店选区的第 10 页的第五格。[/quote]
[quote][color=Blue][b][font=Consolas]i[/font][/b][/color]物品的 slot 值可以通过 商店页数*45+页面内物品格子序号 快速得出。
例如，我想要我的物品位于第三页的第25格，slot 的值即为 3*45+25=160。[/quote]

[size=3][b]堆叠数量（Stack size）：[/b][/size]

[list]
[*]选项 [font=Consolas][b]stack-size[/b][/font] 设置该物品的数量
[*]例如将堆叠数量设置为 5，玩家购买两份物品则会得到 10 个。
[/list]
[code]FoodShop:
  1:
    material: APPLE
    name: "&c20 个苹果"
    buy: 200
    sell: 50
    stack-size: 20[/code]
    
[quote][b][color=Red]！[/color][/b]注意：在设置物品的堆叠数量时，所设置的价格为一整叠物品的价格。这意味着在上述示例中的 10 个苹果向商店购买的价格为 $100，而向商店出售的价格为 $25。[/quote]

[size=3][b]NBT 数据（NBT Data）：[/b][/size]

[list]
[*][font=Consolas][b]NBTData[/b][/font] 是一个用于自定义物品数据的[b]单行[/b]内容
[*]在设置该内容时请参阅 Minecraft 官方维基及其他相关内容来正确配置你的 NBT 数据。
[/list]
[quote][b][color=Red]！[/color][/b]NBTData 设置中若出现英文双引号（"），则最外围需要用英文单引号，例如：'{Enchantments:[{id:"knockback",lvl:50},{id:"sharpness",lvl:50}]}'。
但在包含单引号的 NBTData 中，你需要在每一个单引号处重复输入一个单引号，例如：[font=Consolas]...''{display:{Name:''{"text":"...[/font]][/quote]
[code]#示例 NBT 物品:
NBT_item_shop:
  '1':
    material: potion
    displayname: '&d&l来自 &6金苹果 &d&l的汁液'
    name: '&d&lGod potion'
    buy: 25000
    sell: 2000
    NBTData: '{CustomPotionEffects:[{Id:10,Amplifier:2,Duration:400},{Id:22,Amplifier:4,Duration:2400},{Id:11,Amplifier:1,Duration:6000},{Id:12,Amplifier:1,Duration:6000}]}'
    #若你在 NBT 标签中使用了英文单引号, 你必须输入两个单引号将它 '转义', 这样在代码上它还是会算作一个单引号.
    #例如: 'tags:{customtags:[{''tag'':''thing''}]}'[/code]
[quote][b][color=Red]！[/color][/b]Minecraft 有 255 字符的命令限制，若你输入长于这个限制的命令，超出的部分将会被除去。如果你需要设置长于这个数字的 NBT，请直接打开 YML 文档编辑它。[/quote]

[size=3][b]命令（Commands）：[/b][/size]
[font=Consolas][b][/b][/font]
[list]
[*][font=Consolas][b]buy-commands[/b][/font] 和 [font=Consolas][b]sell-commands[/b][/font] 是玩家在购买或出售后执行的命令列表
[*]对带购买命令的物品使用选项 [font=Consolas][b]giveItem: true[/b][/font] 会在执行命令后得到物品本身
[*]对待出售命令的物品使用选项 [font=Consolas][b]requireItem: true[/b][/font] 将会要求玩家在执行命令前拥有指定的物品
[*]你可以在命令中使用 [font=Consolas][b]%player%[/b][/font]、[font=Consolas][b]%quantity%[/b][/font]、[font=Consolas][b]%buyPrice%[/b][/font]、[font=Consolas][b]%sellPrice%[/b][/font] 等变量来获取交易相关的数据。
[*]以 [font=Consolas][b]p:[/b][/font] 开头的命令为玩家身份执行
[*]以 [font=Consolas][b]o:[/b][/font] 开头的命令为临时管理员身份执行
[/list]
[quote][b][color=Red]！[/color][/b]注意 o: 会给予玩家临时管理员权限，这可能会被恶意利用。强烈图鉴使用例如 LuckPerms 一类的插件给予玩家[b]临时权限[/b]！[/quote]
[code]#示例命令物品:
Buy_commands_item_shop:
  1:
    material: "GOLD_INGOT"
    displayname: "&8&l变得富有或不幸."
    buy: 1000000
    sell: 250000
    giveItem: true
    buy-commands:
    - "/minecraft:give %player% diamond 2"
    - "p: repair hand" # 该命令将会以玩家身份执行
    - "o: i netherite_ingot 5" # 该命令将会以临时管理员身份执行
    requireItem: true
    sell-commands:
    - "eco reset %player%" # 该命令将会以控制台身份执行
    - "effect give %player% minecraft:nausea 255 255"[/code]
[list]
[*][font=Consolas][b]sell-commands[/b][/font] 是玩家向商店出售物品后会执行的命令
[/list]

[size=3][b]无视标签（Ignored tags）：[/b][/size]

这个设置对出售一些特殊物品的情况很有用。

例如在设置中加入 [font=Consolas][b]Enchantment[/b][/font]，则出售的物品无论拥有什么附魔，只要其他条件符合，即可被出售。
如果只添加了这一个标签，物品的标签、名称、描述和其他物品的数据仍然需要与商店中所给的物品完全匹配，才可成功出售。

[quote][b][font=Consolas][color=Blue]i[/color][/font][/b] 这条设置与全局影响 NBT 无视标签设置的功能相同，但只会将作用效果限制在单个物品上。[/quote]

[size=4][b]如何修改语言：[/b][/size]

[list]
[*]在 config.yml 的此处将语言设置为你想要的语言，前提是它存在于你的语言文件中

示例：[font=Consolas]&l<gradient:#F700FF>上一页</gradient:#FFFFFF>[/font]。使用[url=https://www.colorspire.com/rgb-color-wheel/]这个网站[/url]来获取 16 进制的色码。
